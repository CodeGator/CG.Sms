    Remarks

    This method builds up an sms service using information read from the specified configuration.

    Here is an example for the JSON configuration: 

    {
    "Sms": {
      "Strategy": {
          "Name": "Twilio",
          "Assembly": "CG.Sms.Twilio"
        },
      "Twilio": {
      }
   }
}

    Let's break it down. The Services section is where ALL services for the application should be configured. If you're adding a CodeGator service to your application then you'll want to add that service to this section in order to configure the service at startup.

    Under Services, the Sms section is where the CodeGator sms service should be configured. This section contains at least two nodes: Strategy and Assembly. The Strategy node tells the host what strategy to load for the sms service, and as such, is required. The Assembly section is optional, and is only needed when the strategy is located in an external assembly that should be dynamically loaded at startup.

    The Sms section is an example of a strategy section. This will vary, of course, depending on which strategy is named in the Strategy node. In this case, we see a made up example for a Twilio based SMS strategy.
